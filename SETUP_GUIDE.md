# Пошаговая инструкция для новичков

## Шаг 1: Настройка .env файла

1. Откройте файл `.env` в редакторе (Notepad++, VS Code, или даже блокнот)

2. Замените `your_bot_token_here` на ваш токен от @BotFather:
   ```
   BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrsTUVwxyz
   ```

3. Для локальной разработки у вас есть 2 варианта:

### Вариант A: PostgreSQL (рекомендуется для production)

Установите PostgreSQL локально:
- Windows: https://www.postgresql.org/download/windows/
- Или используйте Docker (если установлен)

После установки создайте базу данных:
```sql
CREATE DATABASE task_bot_db;
```

Измените DATABASE_URL в .env:
```
DATABASE_URL=postgresql://postgres:ваш_пароль@localhost:5432/task_bot_db
```
(замените `ваш_пароль` на пароль, который вы установили при установке PostgreSQL)

### Вариант B: SQLite (проще для начала, но не для production)

Можно временно использовать SQLite для тестирования. Но для production нужен PostgreSQL.

## Шаг 2: Установка базы данных

После настройки .env файла выполните:

```bash
npm run db:generate
npm run db:migrate
```

Эти команды создадут структуру базы данных.

## Шаг 3: Запуск бота

Для разработки:
```bash
npm run dev
```

Бот должен запуститься. Вы увидите в консоли что-то вроде:
```
Bot started
```

## Шаг 4: Проверка работы

1. Откройте Telegram
2. Найдите вашего бота (по имени, которое вы дали @BotFather)
3. Отправьте команду `/start`
4. Бот должен ответить приветственным сообщением

## Проблемы?

### Ошибка "BOT_TOKEN is required"
- Проверьте, что в .env файле правильно указан BOT_TOKEN
- Убедитесь, что нет лишних пробелов
- Формат: `BOT_TOKEN=токен_без_кавычек`

### Ошибка подключения к базе данных
- Проверьте, что PostgreSQL запущен
- Проверьте DATABASE_URL в .env
- Убедитесь, что база данных создана

### Бот не отвечает
- Проверьте логи в консоли на наличие ошибок
- Убедитесь, что бот запущен (команда `npm run dev`)
- Проверьте, что вы пишете правильному боту
