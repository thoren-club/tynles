# Нативные возможности для интеграции

Ниже список фич, которые можно добавить, чтобы приложение ощущалось более «нативным».

## Тактильный отклик
- Лёгкие/средние/тяжёлые haptic при ключевых действиях (выполнение задач, пинок).
- Системный fallback на `navigator.vibrate` для браузеров без Telegram Haptic.

## Системные зоны и жесты
- Поддержка `safe-area` (уже используется), чтобы элементы не залезали под вырезы.
- Жесты для шторок/листов: свайп вниз для закрытия, инерция.

## PWA и офлайн
- Установка на главный экран (PWA), собственная иконка и сплэш.
- Офлайн-режим и кэш задач для быстрого старта.

## Быстрые действия и шаринг
- Нативный шаринг (`navigator.share`) для задач/целей.
- Копирование приглашений в буфер (`navigator.clipboard`).

## Системные кнопки
- Telegram WebApp `BackButton` и `MainButton` для системной навигации.
- Управление цветом/темой через Telegram WebApp theme params.

## Биометрия и безопасность
- Блокировка действий с подтверждением через биометрию (если доступно).

## Производительность
- Предзагрузка экранов/данных в фоне (preload).
- Анимации с учётом `prefers-reduced-motion`.
