# Разрешение конфликта Git

Если при `git pull` или `git merge` вы видите ошибку о локальных изменениях, используйте один из вариантов:

## Вариант 1: Сохранить локальные изменения (рекомендуется)

```bash
# Сохранить текущие изменения
git stash

# Обновить из репозитория
git pull

# Применить сохраненные изменения обратно
git stash pop
```

Если после `git stash pop` есть конфликты, разрешите их вручную в файлах.

## Вариант 2: Принять локальные изменения

Если ваши локальные изменения важнее удаленных:

```bash
# Принять локальные версии файлов
git checkout --ours setup-domain.sh
git add setup-domain.sh
git commit -m "Keep local changes"

# Обновить репозиторий
git pull
```

## Вариант 3: Отменить локальные изменения

Если локальные изменения не нужны:

```bash
# Отменить изменения в файле
git checkout -- setup-domain.sh

# Обновить из репозитория
git pull
```

## Вариант 4: Закоммитить изменения

Если хотите сохранить изменения в репозитории:

```bash
# Добавить изменения
git add setup-domain.sh

# Закоммитить
git commit -m "Update setup-domain.sh"

# Обновить из репозитория (может потребоваться merge)
git pull
```

## Быстрое решение (на сервере)

Если вы на сервере и просто хотите продолжить работу:

```bash
# Принять локальные изменения и продолжить
git checkout --ours setup-domain.sh
git add .
git commit -m "Keep local changes" || true
git pull --rebase || git pull --no-edit
```
